# Настройка хостинга для React SPA приложения

При размещении приложения на хостинге с поддержкой подпапок (например, `example.com/app`), необходимо выполнить следующие шаги для исправления ошибки "Not found" при перезагрузке страницы на маршрутах `/records`, `/admin` и др.

## 1. Обновление package.json (если использует подпапка)

Если ваше приложение размещено в подпапке (например, `example.com/app`), добавьте в `package.json`:

```json
"homepage": "/app"
```

или замените `/app` на вашу подпапку.

Если приложение в корне домена, оставьте:
```json
"homepage": "."
```

## 2. Выбор конфигурации по типу хостинга

### Для Apache (самый распространенный)
Файл `.htaccess` уже создан и находится в папке `public/`. При сборке приекта (`npm run build`) он автоматически скопируется в `build/`.

**Убедитесь, что на хостинге:**
- Включен модуль `mod_rewrite`
- Разрешен файл `.htaccess` в корне приложения

### Для Nginx
Если вы используете Nginx, попросите у хостинга или самостоятельно добавьте конфигурацию:

```nginx
location / {
  try_files $uri /index.html;
}
```

### Для Vercel, Netlify, GitHub Pages
Файл `_redirects` уже создан в папке `public/`. На этих платформах он работает автоматически.

### Для других Node.js хостингов (Heroku, etc.)
Если вы используете Express или другой Node.js сервер, убедитесь, что все неизвестные маршруты перенаправляются на `index.html`:

```javascript
app.get('*', (req, res) => {
  res.sendFile(path.join(__dirname, 'public/index.html'));
});
```

## 3. Пересборка приложения

После внесения изменений пересоберите приложение:

```bash
npm run build
```

Затем загрузите содержимое папки `build/` на хостинг.

## 4. Проверка

После развертывания проверьте:
1. Откройте приложение по основной ссылке
2. Перейдите на `/records` или `/admin`
3. Обновите страницу (F5 или Ctrl+R)
4. Если приложение загрузилось без ошибки "Not found" - все работает корректно!

## Доп. информация

- Если остаются проблемы - проверьте консоль браузера (F12 → Console)
- Убедитесь, что на хостинге правильно настроен путь размещения приложения
- Для подпапок может потребоваться перестроить приложение с правильным `homepage` в `package.json`
